# ニュース興味の運用（重みの付け方）

興味は日々変わる前提で、**RSS本文はサーバーがキャッシュ**しつつ、**スコアだけは毎回** Notion の brain（Profile / メモ / アイデア / タスク）と環境変数から再計算します。

## 優先度のイメージ（強い → 弱い）

1. **除外（ミュート）** — タイトルまたは本文に一致した記事は落とす  
2. **朝の感想メモ `[ニュースFB]`** — チャットで一言いただいた「もっと／減らしたい」を構造化して Memos に保存し、**次回のスコアに強めで反映**（新しい FB メモほど効く）  
3. **Notion Profile の明示ルール** — 「ニュース関心」行の本文（カンマ区切りフレーズ）  
4. **`KAGE_NEWS_KEYWORDS`** — デプロイ単位のベース興味（`KAGE_NEWS_ENV_WEIGHT` で倍率）  
5. **弱いシグナル**（任意）— メモ・アイデア・タスクのタイトルから短い語を拾う（`KAGE_NEWS_MINING_PROFILE=1` で Profile 本文も弱くマイニング）

### 朝の感想フロー（運用）

1. 起動時に `/opening?bootstrap_session=1` でセッションが作られ、`/morning?session_id=…` のあと **ニュース付き**なら UI が一言感想を促す。  
2. ボスが返信すると、サーバが **Gemini で JSON 化**（`more` / `less` / `skip`）し、Notion Memos に **`[ニュースFB] YYYY-MM-DD HH:MM`** として保存。  
3. **「特にない」** 等のスキップは保存せず丁寧に終了。  
4. **予定確認っぽい文**（「今日の予定は？」等）は誤爆しないよう、感想フローに吸い込まない。  
5. セッションの `pending` は **`KAGE_NEWS_FEEDBACK_TTL_SEC`**（既定48h）で自然失効。

メモ本文の JSON 例: `{"more":["生成AI","教育DX"],"less":["株"],"brief":"…","user_raw":"…"}`

## Notion Profile の書き方

| 名前（タイトル）の例 | 内容（本文） |
|---------------------|--------------|
| `ニュース関心` または `ニュース:関心` | `RAG, 生成AI, 広告クリエイティブ, 教育DX` のようにカンマ区切り |
| `ニュース除外` または `ニュース:除外` | `株価, 芸能, 野球` など、出したくない語句 |

- **カテゴリ**が `ニュース` の行は、本文を **関心フレーズ** として扱います（タイトルが上記プレフィックスでなくても可）。
- Profile は API 上 **最終更新が新しい順** に並び、**上の行ほど減衰が小さい**重みになります（最近いじったルールが効きやすい）。

## 環境変数

| 変数 | 役割 |
|------|------|
| `KAGE_NEWS_KEYWORDS` | 常に効かせたいキーワード（カンマ区切り） |
| `KAGE_NEWS_MUTE_KEYWORDS` | 全記事に対する除外語（小文字化して部分一致） |
| `KAGE_NEWS_ENV_WEIGHT` | 環境キーワードの係数（既定 `1.0`） |
| `KAGE_NEWS_MINING_PROFILE` | `1` で Profile 一般テキストから弱い語も抽出（ノイズ増のため既定オフ推奨） |
| `KAGE_NEWS_CACHE_SEC` | RSS **生データ**の再取得間隔（スコアはキャッシュと独立して毎回再計算） |

## 日々の運用のコツ

- 関心が変わったら **Profile に1行追加 or 既存の「ニュース関心」を編集**（編集日が新しくなるほど強く効く）。  
- 一時的にうるさいトピックは **「ニュース除外」** に足す（コードデプロイ不要）。  
- Cron で `GET /news/digest?refresh=1` を叩くのは **RSSの中身を更新**するため。興味の変化だけなら通常の `/morning` で十分反映されます。

## API レスポンスの `interest` フィールド

`GET /morning` の `news.interest` および `GET /news/digest` の `interest` に、デバッグ用に次が入ります。

- `top_terms` — 重み付き後に強い語（先頭数十語）  
- `sources` — どのソースから語が増えたかの粗いカウント  
- `mutes_count` — 除外語の件数  

詳細は `news_digest.merge_weighted_signals` を参照。
