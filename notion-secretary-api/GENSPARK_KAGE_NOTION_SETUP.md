# Genspark / 作業代行向け：KAGE 議事録 × Notion セットアップ指示書

この文書をそのまま貼り付け、**Notion の画面操作と Railway の環境変数**を完了させてください。  
（Genspark がブラウザ操作できる前提。できない場合は人間が手順どおり実施。）

---

## 目的

- KAGE（`notion-secretary-api`）が **議事録** を Notion に保存できるようにする。
- **長文**はサーバ側で **要約**し、**原文も残す**（列の設定次第で「内容」と「原文」に分離可能）。

本番例（参考）: `https://notion-secretary-api-production.up.railway.app/app`

---

## 前提

- ワークスペースに **Go_KAGE** まわりの DB がある（Schedule / Tasks / Memos 等）。
- **KAGE 用 Notion インテグレーション**（例: 名前が `Go_KAGE`）が既に存在し、他 DB に接続済みである。
- **Railway** に `notion-secretary-api` がデプロイされ、`NOTION_API_KEY` / `GEMINI_API_KEY` が設定済みである。

---

## 作業 1：Notion に「議事録」データベースを用意する

### 1-1. 置き場所

- **プライベート** → **Go_KAGE** と同じ階層に、他 DB（会話ログ・デバッグログ等）と**並ぶ**形で配置する。

### 1-2. データベース名

- タイトル例: **`議事録`**

### 1-3. 必須プロパティ（列）

| 表示名 | 種類 | 備考 |
|--------|------|------|
| **名前** | **タイトル** | 会議名。DB のメイン列は必ず **タイトル** 型（**テキスト**型にしない）。 |
| **日時** | **日付** | 会議日または開始日時。 |
| **内容** | **テキスト**（または長いテキスト） | **要約**や結合保存が入る。**タイトル** 型の列にしない（API は本文を rich_text で送る）。 |

### 1-4. 推奨プロパティ（列分けしたい場合）

| 表示名 | 種類 | 備考 |
|--------|------|------|
| **原文** | **長いテキスト** | 貼り付けた**生テキスト全文**。あると要約と分離できる。 |

- **原文列を付けない場合**でも動作する（要約＋原文が **`内容` 1 列**に `## 要約` / `## 原文（未加工）` で入る）。

### 1-5. 列の型を取り違えないこと（400 エラーの典型）

- **`名前` がテキスト**、**`内容` がタイトル** になっていると、保存時に  
  `名前 is expected to be rich_text` / `内容 is expected to be title` のような API エラーになる。
- **直し方（推奨）**: Notion で列の **種類** を上表どおりに揃える（既存列の種類は変更できない場合は、正しい種類で **新しい列を追加**し、誤った列は削除または非表示）。
- **Railway の `NOTION_MINUTES_TITLE_PROP` / `NOTION_MINUTES_CONTENT_PROP` で列名を入れ替えて対応する方法**もあるが、**手順どおりのスキーマに直す方がねじれがない**。

### 1-6. メイン列（名前）について

- メイン列の種類を **テキスト** に変更しない（**タイトル** のまま）。

## 作業 2：インテグレーションを接続する

1. **`議事録` データベースのページ**を開く（表ビューでよい）。
2. 右上 **`⋯`** → **`コネクトを追加`**（接続）。
3. 既存の **KAGE / Go_KAGE 用インテグレーション**を選び、**この DB に追加**する。
4. **読む・挿入・更新**が許可されていることを確認する。

---

## 作業 3：データベース ID を取得する

1. **`議事録` をフルページで開いた状態**のブラウザ **URL** をコピーする。
2. 形式例:  
   `https://www.notion.so/32cc70f70203805dbfb8e945ad117bda?v=...`
3. **`?` より前**の、`notion.so/` の直後の **32 文字前後の ID** だけを取り出す。  
   - **悪い例**: `v=` 以降（ビュー ID）をコピーしない。  
   - **良い例**: `32cc70f70203805dbfb8e945ad117bda`  
   - ハイフン付き UUID に整形してもよい（Railway はどちらでも多くの場合動く）。

## 作業 4：Railway の環境変数

対象: **notion-secretary-api を動かしているサービス**の **Variables**（プロジェクト共通だけに付けない）。

| Variable | 値 |
|----------|-----|
| **`NOTION_DB_MINUTES`** | 手順 3 でコピーした **データベース ID**（前後にスペースなし） |

作業 1 の **名前＝タイトル / 内容＝テキスト** どおりなら、**`NOTION_MINUTES_*` の列名用変数は設定不要**です。KAGE は **`NOTION_DB_MINUTES` だけ**で Notion API から DB のプロパティ一覧を取得し、**title / date / rich_text を型で自動対応付け**します（列名が「会議名」「本文」などでも、型さえ合っていれば拾います）。

### 任意（列分けする場合）

| Variable | 値 |
|----------|-----|
| **`NOTION_MINUTES_RAW_PROP`** | `原文`（Notion の列名と**完全一致**） |

### 任意（要約のオンオフ・閾値）

| Variable | 説明 |
|----------|------|
| `KAGE_MINUTES_SUMMARIZE` | `1` 有効（既定） / `0` で要約しない |
| `KAGE_MINUTES_SUMMARIZE_THRESHOLD` | この文字数以上で要約（既定 `1200`） |

保存後、**再デプロイ**が走るまで待つか **Redeploy** する。
